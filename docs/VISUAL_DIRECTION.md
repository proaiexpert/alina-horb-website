## Implementation rules

- Reproduce the approved visual direction faithfully rather than replacing it with standard cards or a conventional header.
- Responsive layouts may reflow, but must preserve hierarchy, editorial character and whitespace.
- Avoid heavy shadows, glassmorphism, gradients, neon, generic icon grids and oversized rounded cards.
- Body copy must remain readable at production scale.
- The approved mockup is a visual target, not permission to rasterize the page.
- All text, controls and layout must be implemented as accessible HTML/CSS components.
